# 报警根因分析 (alarm-root-cause-analysis)
基于 DCS/GDS/SIS 实时数据，运用规则引擎与大模型混合推理，自动诊断化工生产过程中报警信号的根因。
## 触发条件
  • 单点报警：GDS 报警信号触发后 30s 内自动启动
  • 关联分析：同一工段 5min 内出现 3+ 报警时自动升级为关联模式
  • 手动调用：数字员工通过对话或任务下发触发
## 执行步骤
Step 1 - 数据采集
  • 从 DCS 获取报警点位实时数据（最近 15min）
  • 从 GDS 获取气体检测浓度趋势
  • 从 SIS 获取联锁状态
Step 2 - 规则预筛
  • 匹配 已知规则库（200+ 条预置规则）
  • 规则命中则直接输出结论，置信度 ≥ 0.95
  • 规则未命中则转大模型推理
Step 3 - 大模型推理
  • 将报警上下文组装为结构化 Prompt
  • 调用诊断模型（128K 上下文窗口）
  • 输出根因假设 + 置信度评分
Step 4 - 报告生成
  • 生成结构化诊断报告
  • 附处置建议、风险提示、历史对比
  • 推送至任务中心归档
## 输出格式
诊断报告包含：根因结论、置信度、处置建议、历史同类报警、关联技能推荐
## 限制与约束
  • 单次 Token 消耗上限：8K
  • 推理超时：30s
  • 置信度低于 0.6 时标记为"需人工确认"
